## 2. Simulated Auditor Q&A (Mock Audit)

**Auditor:** *"How do you ensure that developers don't have permanent root access to the production database?"*

**NexaCore Response:**
"We follow the **Principle of Least Privilege (A.8.3)**. Access is managed via AWS IAM Groups. Developers have Read-Only access in Staging. For Production emergencies, access is granted via a 'Break-Glass' procedure which triggers a CloudWatch Alert to the Security Lead."

## 3. Continuous Monitoring
NexaCore uses **AWS Security Hub** and **GitHub Dependabot** to provide real-time evidence of our compliance posture. Weekly vulnerability reports are generated and reviewed by the Security Lead.
